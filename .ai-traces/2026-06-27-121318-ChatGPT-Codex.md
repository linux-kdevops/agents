## Problem Analysis

MACP tracks agent sessions, handoffs, work items, prompts, and retrieved
memory, but it did not answer whether an agent task had the right reusable
skill. The OpenSkill paper frames skill improvement as an explicit artifact
loop: acquire open-world evidence, write or refine skills, verify them with
independent virtual tasks, and reserve target supervision for final evaluation.

For MACP, the practical need is a lightweight way to make skill gaps visible,
measure skill coverage over time, and keep quality claims tied to checks.

## Considered Approaches

1. **Commit trailers only**: easy to validate, but hard to aggregate across
   tasks and poor for repeated eval results.

2. **Ledger only**: easy to compute metrics, but commits would not clearly
   bind a code/doc change to the skill event that motivated it.

3. **Ledger plus optional trailers**: selected because the ledger is the
   durable metric source and trailers provide commit-level provenance without
   changing the base MACP format.

## Implementation Decisions

- Added a `SKILLS.md` append-only TSV block inside Markdown so humans can
  review it and scripts can parse it without dependencies.
- Added optional `Skill-*` trailer validation to the base and lite hooks.
  Validation fires only when trailers are present.
- Added `bin/skill-ledger-metrics.py` to compute skill coverage, open gaps,
  eval scores, lift, verifier coverage, and cost.
- Documented three eval levels: smoke, virtual, and final/holdout. Virtual
  checks may guide skill edits; final checks measure outcome and create
  follow-up tasks, but hidden details stay out of skills.
